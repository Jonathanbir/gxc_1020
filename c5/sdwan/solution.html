    • SD-WAN Service

    中華電信以 SD-WAN 技術結合優質網路及服務資源，為客戶提供頻寬使用更彈性、建置維運更方便、網路品質更優化的智慧化網路解決方案

    • 產品總覽

    • 服務介紹

    • 特色與優勢

    • 解決方案

    • 情境一

      簡化管理與設定

    • 情境二

      應用分流及提升服務品質

    • 情境三

      提升使用公有雲的傳輸品質

    網路環境

    • 客戶總部在台灣
    • 海外有五個據點，三個據點分布於亞太，美國及歐洲各一據點
    • 總部及海外據點皆具有 MPLS IPVPN 及 Internet
    • 總部 MPLS IPVPN 頻寬 30Mbps，Internet 100Mbps
    • 三個亞太據點 MPLS IPVPN 頻寬 10M，Internet 50Mbps
    • 美國及歐洲據點 MPLS IPVPN 頻寬 5M，Internet 20Mbps

    問題與解決方案

    客戶以 MPLS IPVPN 及 Internet 作分流，設定複雜

    • SD-WAN 提供 Web 圖形化設定模式，方便於集中介面進行全網的視覺化管理及設定。

    當網段異動或新增服務據點時，須分別連入各據點做相對應的調整，造成管理人員很大的負擔

    • SD-WAN集中控管元件為 SD-WAN 最重要指揮中心，必須確保其網路及設備的可用性，中華電信於骨幹建置 SD-WAN 集中控管元件，免去客戶自建自維的負擔。

    當網管人員異動時，新接手之網管人員難以承接

    • 中華電信提供 SD-WAN 的託管服務，深入了解客戶網路環境及使用需求，協助客戶進行網路規劃、設定、管理。
    • 中華電信提供完善的網管系統。
    • 7 x 24 Managed Service Center 專業維護人員，提供中英文服務。
    • MPLS IPVPN 專線路徑
    • 全段 Internet 路徑

    網路環境

    • 客戶總部在台灣，海外有兩據點，每一據點都有 Internet 及 MPLS IPVPN 服務，MPLS IPVPN 頻寬比 Internet 頻寬相距很大
    • 將使用頻寬較小且重用的應用以 MPLS IPVPN 為主電路：ERP、語
    • 將使用頻寬較大之應用 ( 如視訊、檔案傳輸等 ) 以Internet IPSec為主電路

    問題與解決方案

    傳統路由器須當線路完全中斷才進行切換，造成應用的傳輸品質不良及重要服務 downtime 時間過久

    • SD-WAN 可依據 SLA (Packet loss、Jitter、Latency) 設定網路切換的條件，提升應用之傳輸品質及降低 downtime。

    訊視IP經常異動，分流設定亦須做相對應的調整，管理不易

    • SD-WAN可依據應用層分流，故可設定當 Traffic 的 pattern 為視訊時透過 Internet IPSec 或 MPLS IPVPN 傳輸，而不需要以 IP 決定分流路徑。

    當Internet切換至 MPLS IPVPN 時，因為可用頻寬縮小造成重要應用通訊品質不良

    • 中華電信設計完善的 SD-WAN end-to-end Qos 機制，並具備豐富的 Qos 規劃經驗，於頻寬壅塞時保障重要應用傳輸品質。
    • MPLS IPVPN 專線路徑

      ERP、Voice主線路
      Video 備援電路

    • 全段 Internet 路徑

      Video 備援電路
      ERP、Voice主線路

    網路環境

    • 客戶總部在台灣
    • 三個 Branch 分別位於台灣、越南、大陸
    • 客戶使用 AWS 服務，AWS 的 VPC (Virtual Private Cloud) 建置於日本，各據點皆有存取日本 VPC 的需求

    問題與解決方案

    透過網際網路品質存取AWS的品質不易掌控，且可能遇到封網問題

    • 中華電信骨幹連結各主要公有雲服務。
    • 客戶利用多種網路就近連結中華電信 SD-WAN 骨幹節點，透過中華電信骨幹連結各公有雲服務。
    • MPL 透過 MPLS IPVPN專線存取 AWS
    • 經 Region Internet 進入中華 SD-WAN 局端設備存取 AWS
